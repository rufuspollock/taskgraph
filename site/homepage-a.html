TaskGraph

Capture fast. Keep markdown. Query the graph.

TaskGraph is a local-first CLI for quick task capture and simple task listing. Markdown stays the source of truth, SQLite stays derived, and AI tools get a clean planning substrate instead of another silo.

taskgraph demo local

Markdown files remain authoritative. The SQLite index exists to make listing and graph views fast.

Install

Use the release installer on macOS or Linux. It supports amd64 and arm64 and installs to ${INSTALL_DIR:-$HOME/.local/bin} by default.

latest release

curl -fsSL https://raw.githubusercontent.com/rufuspollock/taskgraph/main/scripts/install.sh | bash

Then run tg init in a project and start capturing tasks.

Why TaskGraph

  • Capture tasks quickly with tg add and keep moving.
  • Store authoritative task data in markdown, not in a hidden app database.
  • Use a derived SQLite index for fast list and graph-style queries.
  • Work naturally with AI tools that need local, inspectable planning context.

Capture

Add tasks into .taskgraph/issues.md with labels and types stored inline as markdown tags.

List

Query the indexed task set with filters like --label without giving up markdown as the source of truth.

Graph

See a compact overview of real roots and child structure with tg graph.

See what it can do

A lightweight workflow from quick capture to graph-native planning. 

tg init
tg add "buy milk"
tg add "plan flower show" --labels flowershow,events
tg add "map launch dependencies" --type epic
tg create "book dentist"
tg inbox
tg list --label flowershow
tg graph --depth 3

What this gives you

Fast inbox capture without committing to a heavyweight task system.

One local task substrate that can span scattered markdown files and checklist hierarchies.

A practical base for future graph-native planning commands like tg next.

Local-first by design

TaskGraph keeps markdown authoritative and treats the database as rebuildable derived state. That keeps the system inspectable, scriptable, and hard to get trapped inside.
